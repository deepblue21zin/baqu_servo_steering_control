# 조향 제어 시스템 통합(Pulse Control)

> **작성일:** 2026년 3월 8일  
> **프로젝트:** `servo_control_baqu` (STM32 NUCLEO-F429ZI + LS L7S 서보 드라이브)  
> **목적:** 이더넷(UDP) 기반 1ms 실시간 조향 제어 시스템의 구조 파악 및 실무 지침서  

---

## 1. 시스템 개요 및 구동 방식 (System Overview)


본 시스템은 자율주행 인지 파트(PC) 또는 수동 조이스틱(ASMS)으로부터 조향 명령을 받아, 서보 모터를 1ms의 엄격한 데드라인 내에 제어하는 임베디드 소프트웨어입니다.

### 데이터 흐름 (Data Flow)
1. **수신 (인지):** LwIP를 통해 UDP 포트 5000번으로 목표 조향각 데이터 수신
2. **목표 설정:** `main.c` 루프에서 데이터를 파싱하여 PID 제어기의 Target Angle로 설정
3. **현재 상태 파악:** 1ms마다 TIM4 하드웨어 타이머가 엔코더(Encoder) 펄스를 읽어 현재 각도 파악
4. **제어 연산:** 현재 각도와 목표 각도의 오차를 바탕으로 PID 연산 수행 (목표 속도/주파수 산출)
5. **동작 수행 (구동):** 산출된 주파수를 바탕으로 TIM1 PWM 펄스를 생성하여 모터 회전 (Pulse Control)

## 2. 핵심 모듈 상세 분석 (Core Modules)

### 2.1 통신 모듈 (`ethernet_communication.c` & `lwip.c`)
* **역할:** 자율주행 PC 및 ASMS와 통신하며 데이터를 수신.
* **통신 방식:** UDP 브로드캐스트 (빠른 전송 속도 및 데이터 손실 시 다음 패킷으로 즉시 대체).
* **데이터 필터링:** * **PC (자동):** 데이터 길이 9 바이트, 발신자 IP `.1` -> `AUTO` 모드 시 조향각 반영
  * **ASMS (수동):** 데이터 길이 5 바이트, 발신자 IP `.5` -> `MANUAL` 모드 시 조향각 반영
* **Fail-safe (통신 안전):** 300ms 동안 패킷이 들어오지 않으면 즉시 `ESTOP`(비상정지) 상태로 전환.
* **주의사항:** RTOS가 없으므로 `main.c`의 무한 루프에서 `MX_LWIP_Process()`를 지속적으로 폴링(Polling)해야 수신 가능.

### 2.2 메인 제어 및 타이머 (`main.c` & `stm32f4xx_it.c`)
* **역할:** 시스템 초기화, 1ms 제어 루프 타이밍 제공.
* **SysTick 인터럽트:** 1ms마다 `SysTick_Handler()`가 실행되어 `interrupt_flag = 1`로 세팅.
* **동기화 보장:** 공유 변수(목표 각도 등) 수정 시 `__disable_irq()`와 `__enable_irq()`를 사용하여 임계영역(Critical Section)을 보호하고, `volatile` 키워드로 변수 최적화 방지.

### 2.3 위치 제어 두뇌 (`position_control.c`)
* **역할:** 1ms 주기로 `PositionControl_Update()`를 실행하여 오차를 줄이는 방향으로 연산.
* **PID 연산:** $Output = K_p e(t) + K_i \int e(t) dt + K_d \frac{de(t)}{dt}$
* **안전 점검:** 모터가 물리적 한계(±4320도, 12회전)를 넘거나 오차가 너무 크면 즉시 비상정지.

### 2.4 근육과 신경: 담당 D (바쿠) 영역

* **엔코더 (`encoder_reader.c`):** * TIM4(16비트)를 사용하며, 오버플로우를 막기 위해 카운터 중간값인 **32768**부터 시작하도록 보정.
  * 0.0075도 분해능으로 현재 위치 정밀 파악.
* **펄스 제어 (`pulse_control.c`):** * `PulseControl_SetFrequency()` 함수가 방향 핀(DIR)을 설정하고, TIM1의 ARR 레지스터 값을 조작하여 PWM 주파수 출력.
  * 서보 드라이브로 직접 구형파 신호를 쏘아 모터를 회전시킴.

## 4. 핵심 변경 이력 및 트러블슈팅 (History & Fixes)

1. **TIM2 -> TIM4 이관 (PA1 핀 충돌):** 이더넷(RMII) 필수 핀인 PA1이 기존 TIM2 엔코더와 겹쳐서비 엔코더를 TIM4로 이동. 16트 타이머의 한계는 32768 오프셋 보정으로 해결.
2. **DIR 핀 정합화 (PE11 -> PE10):** 코드와 문서 간 방향 제어 핀 정보가 불일치했던 것을 **PE10**으로 통일.
3. **진정한 Emergency Stop:** 소프트웨어 정지뿐만 아니라 `Relay_Emergency()`를 추가 호출하여 하드웨어 릴레이(PD15) 전원을 끊도록 2중 안전 확보. 
4. **다회전 운영 정책 반영:** 12:1 기어비를 고려하여 조향 허용 범위를 ±360도에서 ±4320도로 대폭 확장.

## 5. 지연 시간 (Latency) 측정 시스템 적용

조향 제어가 1ms의 데드라인을 완벽히 지키는지 증명하기 위한 측정 시스템이 도입되었습니다.

* **측정 원리:** STM32 코어의 하드웨어 사이클 카운터(DWT CYCCNT)를 이용하여 180MHz 클럭 기준으로 마이크로초(us) 단위 정밀 시간 측정.
* **측정 구간 (Stage):** * `ID1 Sense`: 엔코더 읽기
  * `ID2 Control`: PID 연산
  * `ID3 Actuate`: **펄스 제어 (바쿠 담당 구역)**
  * `ID4 Comms`: 이더넷 통신
* **데드라인 미스 감지:** `SysTick_Handler`에서 이전 1ms 주기의 작업이 아직 안 끝났다면 `deadline_miss_count` 증가.
* **데이터 로깅:** 2,000 샘플마다 평균(avg), p99, 최대(max) 시간을 UART로 자동 출력하며, Windows PC에서 스크립트를 통해 CSV 형태로 증거 자료 저장.

## 6. 인지 사항 및 미션

네가 작성하고 최적화해야 하는 `Pulse Control` 파트는 **시스템 전체의 실시간성을 좌우하는 가장 중요한 부분**이야.

1. **시간 예산 준수 (Actuate 지연 방지):** 네가 담당하는 `ID3 Actuate` 구간의 연산 시간은 **최대 0.20ms (200us)**를 넘으면 안 돼. `PulseControl_SetFrequency` 함수 내부에 무거운 연산이나 반복문을 넣지 마.
2. **로그 출력 통제:** UART `printf` 출력은 속도를 엄청나게 깎아먹어. 측정 구간 안에서는 로그를 빼거나 반드시 `#if LATENCY_LOG_ENABLE` 매크로로 감싸서 실제 운영 빌드에서는 꺼지도록 만들어야 해.
3. **오버플로우 방지:** 엔코더(TIM4)가 16비트이기 때문에 값이 누적되면서 튀지 않도록 delta 계산 및 unwrap 로직을 안정적으로 유지해야 해.
4. **방향 제어 핀 주의:** 하드웨어 조작 시 방향 핀은 무조건 `PE10`임을 기억하고 결선 오류를 방지할 것.
