# Fix Log - 2026-03-05

## 요약

오늘 변경의 목적은 다음 3가지였다.

1. 조향 지연 측정을 코드에 실제 삽입 (DWT, p99/worst, deadline miss)
2. 2000샘플 자동 배치 출력 + Windows(PuTTY) 로그 자동 저장 체계 구축
3. 산업 운용 관점 보강 (fail-safe 주석 정리, watchdog 타임아웃 조정, 운영 한계값 분리)

---

## 코드 변경

### 1) Latency 계측 모듈 신규 추가

- `Core/Inc/latency_profiler.h` (신규)
- `Core/Src/latency_profiler.c` (신규)

주요 내용:
- DWT CYCCNT 초기화
- 구간별 계측 매크로 `LAT_BEGIN/LAT_END`
- stage 통계 `avg/p99/max` 계산
- `deadline_miss_count` 카운트

### 2) 메인 루프 계측/자동 리포트

- `Core/Src/main.c`

주요 내용:
- `LatencyProfiler_Init(SystemCoreClock)` 호출
- `Comms` 구간 계측 삽입
- 2000샘플 도달 시 자동 CSV 배치 출력:
  - `LATENCY_BATCH_BEGIN`
  - `LATENCY_STAGE`
  - `LATENCY_BATCH_END`
- 배치 출력 후 `LatencyProfiler_Reset()`으로 다음 배치 반복 수집

### 3) 제어 루프 계측

- `Core/Src/position_control.c`

주요 내용:
- `Sense/Control/Actuate` 구간 계측 삽입
- 측정 중 로그 영향 분리를 위한 `LATENCY_LOG_ENABLE` 분기 적용
- `EmergencyStop` 주석을 실제 동작(EMG 릴레이 정지 수행)과 일치하도록 수정

### 4) deadline miss 계수

- `Core/Src/stm32f4xx_it.c`

주요 내용:
- `SysTick`에서 이전 tick 미처리 여부를 `LatencyProfiler_OnDeadlineTick()`으로 집계

### 5) Watchdog 안전값 조정

- `Core/Src/iwdg.c`

변경:
- 기존: `Prescaler=256`, `Reload=4095` (매우 긴 타임아웃)
- 변경: `Prescaler=64`, `Reload=249` (약 0.5초)

의도:
- 제어 시스템 소프트락/멈춤을 더 빠르게 감지

### 6) Ethernet 모듈 정리 (산업 운용 가독성)

- `Core/Inc/ethernet_communication.h`
- `Core/Src/ethernet_communication.c`

변경:
- `ETHCOMM_LOG_ENABLE` 로그 스위치 추가
- `ETHCOMM_LOG(...)` 매크로로 로그 통일
- Legacy text-command API를 "벤치/디버깅용"으로 명시
- 현재 런타임 경로가 UDP 기반임을 파일 상단 주석에 명시

### 7) 운영 한계값(Operational Limit) 분리 적용 (이후 Step 8에서 철회)

- `Core/Src/main.c`

추가:
- 물리 한계: `STEER_LIMIT_PHYSICAL_DEG = 360`
- AUTO 운용 한계: `STEER_LIMIT_AUTO_DEG = 120`
- MANUAL 운용 한계: `STEER_LIMIT_MANUAL_DEG = 360`
- 목표값 적용 시 mode별로 clamp 후 `PositionControl_SetTarget()` 호출

비고:
- 해당 정책은 같은 날 12:1 다회전 운용 요구로 철회됨.
- 최종 반영 상태는 아래 Step 8 기준.

### 8) 다회전(12:1 기어비) 운용 반영으로 한계 정책 재조정

- `Core/Src/main.c`
- `Core/Inc/position_control.h`
- `Core/Src/position_control.c`
- `Core/Src/ethernet_communication.c`

변경:
- `main.c`의 AUTO/MANUAL 모드별 운영 clamp 제거
- `PositionControl_SetTarget()` 허용 범위를 `±360` → `±4320`으로 확대
- `MAX_TRACKING_ERROR_DEG = 4500` 추가 (기존 150도 임계 대체)
- Ethernet 수신 clamp를 고정 360이 아닌 `MAX_ANGLE_DEG/MIN_ANGLE_DEG` 기반으로 통일

의도:
- 모터 다회전 명령(12회전=4320deg) 운용 시 정상 추종 가능하도록 제어/안전 범위 일치

```c
hiwdg.Init.Reload = 249;   // 약 0.5초
```

### B) EmergencyStop 주석 정합화 (`Core/Src/position_control.c`)

```c
// Before (comment)

// After (comment)
// 현재는 소프트 정지 + EMG 릴레이 정지를 함께 수행.
```

### C) Ethernet 로그 정책 통일 (`Core/Src/ethernet_communication.c`)

```c
// Before
printf("[EthComm] ERROR: udp_new() failed\\r\\n");

// After
ETHCOMM_LOG("[EthComm] ERROR: udp_new() failed\\r\\n");
```

### D) 다회전(12:1) 운용 범위 확장 (`Core/Inc/position_control.h`)

```c
// Before
#define MAX_ANGLE_DEG   360.0f
#define MIN_ANGLE_DEG  -360.0f

// After
#define MAX_ANGLE_DEG   4320.0f
#define MIN_ANGLE_DEG  -4320.0f
#define MAX_TRACKING_ERROR_DEG 4500.0f
```

### E) 모드별 운영 clamp 제거 (`Core/Src/main.c`)

```c
// Before
float op_limit = (mode == STEER_MODE_AUTO) ? STEER_LIMIT_AUTO_DEG : STEER_LIMIT_MANUAL_DEG;
float safe_target = clamp_target_deg(pkt.steering_angle, op_limit);
PositionControl_SetTarget(safe_target);

// After
PositionControl_SetTarget(pkt.steering_angle);
```
