Project Goal

조향 서브컨트롤러가 실제로 해야 하는 일

프로젝트 목표

이 프로젝트의 역할은 상위 제어기에서 받은 조향축 기준 목표값을 STM32가 1ms 폐루프 제어와 fail-safe를 통해 실제 조향 액추에이터에 안전하게 전달하는 것이다.

  • 외부 명령을 `steering_deg` 기준으로 받고 내부에서는 `motor_deg / encoder_count / pulse`로 변환한다.
  • 센서, 제어기, 액추에이터, 릴레이, 통신, 로그 증거까지 하나의 runtime으로 묶는다.
  • 실동작과 함께 latency, fault, traceability를 수치로 남긴다.
  • 자율주행 조향 기준으로 startup, homing, recovery, ESTOP 계약을 설명 가능해야 한다.

Data Contracts

받고 보내는 정보 종류와 규격

현업에서는 기능보다 단위와 계약이 먼저 보인다. 이 프로젝트는 현재 `steering_deg`와 `motor_deg`가 혼재되어 있으므로, 아래 표를 기준으로 정리해야 한다.

Producer Consumer Payload Current Format Recommended Contract
`ethernet_communication.c` `main.c` `AutoDrive_Packet_t.steering_angle` `float` `steering_deg`, clamp `[-45, +45]`
`main.c` `position_control.c` target `float` `motor_deg`, converted via `SteeringDegToMotorDeg()`
`encoder_reader.c` `position_control.c` current angle `float` `motor_deg` + raw/accum count 함께 제공
`position_control.c` `pulse_control.c` actuator command `int32_t` signed `pulse_hz`
`position_control.c` `relay_control.c` emergency request function call latched fail-safe event + reason code
`main.c` / `position_control.c` UART / plots / CubeMonitor `CSV`, `LATENCY_STAGE`, `dbg_*` text + global vars traceability artifact set

저장되는 로그 종류

  • `CSV_HEADER`, `CSV,...` : target / current / error / output / dir / enc
  • `CMD_START`, `CMD_REACHED`, `CMD_TIMEOUT`, `CMD_ABORT`, `CMD_FAULT` : 명령 단위 lifecycle event
  • `LATENCY_BATCH_BEGIN`, `LATENCY_STAGE`, `LATENCY_BATCH_END` : Sense, Control, Actuate, Comms timing
  • `[KB][target]`, `[KB][snapshot]` : bench interaction snapshot
  • `dbg_*` globals : CubeIDE watch, CubeMonitor, UART 분석용
  • `Doc/change_code/YYYY-MM-DD.md` : 설계/코드 변경 traceability

현업식 분석 흐름

  1. Upper command, target unit, gear ratio, pulse conversion이 같은 기준인지 먼저 확인한다.
  2. 센서값이 실제 변하는지 `enc_raw`, `current_deg`, `ADC angle`부터 확인한다.
  3. 그 다음에 `error`, `output`, `dir`, `pulse`를 비교한다.
  4. 마지막에 latency, deadline miss, fail-safe trace를 붙여 deterministic behavior를 설명한다.

Latency evidence

measured

현재 로그에서 확인되는 사실

  • control samples 154개에서 `enc_raw` unique 값이 1개뿐이라 current feedback path가 검증되지 않았다.
  • 대신 `CMD_START/REACHED/TIMEOUT/ABORT/FAULT` 이벤트가 추가되어 명령 단위 추적 기반은 갖춰졌다.
  • `DIR`은 로그상 0/1로 변하므로 MCU 내부 direction GPIO는 바뀌고 있다.
  • latency는 대체로 낮지만 interactive logging 구간에서 Comms max가 12 ms 이상 튀는 outlier가 있다.
  • deadline miss는 전체 220 batch 중 non-zero batch가 존재하므로 strict real-time claim에는 조건 설명이 필요하다.

분석 절차

  1. UART raw log를 PuTTY session log로 저장한다.
  2. `python putty_log/plot_putty_log.py ...`로 control / latency plot을 생성한다.
  3. `target`, `current`, `error`, `output`, `dir`, `enc`를 먼저 본다.
  4. 그 다음 `LATENCY_STAGE`와 deadline miss를 붙여 timing evidence를 만든다.
  5. 이 결과를 `Doc/change_code`, 팀원 REQ, Doxygen source와 연결해 traceability를 만든다.

Industry Evaluation

현업 면접관 시점의 프로젝트 완성도 평가

여기서 100%는 자율주행 조향 서브컨트롤러로서 production-complete한 상태를 의미한다. 즉 코드가 돌아가는 수준이 아니라, 센서 진실성, command lifecycle, fail-safe, startup safety, 반복 검증, traceability까지 모두 설명 가능한 상태를 기준으로 본다.

Current Problems

지금의 문제와 왜 중요한가

P0. Encoder feedback stuck

latest measured logs에서 `enc_raw`가 `32769`로 고정된다. 따라서 현재 상태는 “closed-loop steering validated”라고 부를 수 없다.

Evidence: unique encoder values = 1

P1. System readiness closure incomplete

`command_id`와 `CMD_START/REACHED/TIMEOUT/ABORT/FAULT`는 추가됐지만, boot-time auto-enable, latched fault, startup gating, sensor cross-check까지는 아직 닫히지 않았다.

Evidence: Need ready contract + fault latch + clear policy

P1. 16-bit unwrap missing

`±45 deg steering`를 커버하려면 `±72000 count`가 필요한데, 현재 centered 16-bit raw 방식은 full range 보장이 어렵다.

Need: accum_count / wrap tracking

P1. Boot-time auto enable

main runtime에서 homing/state validation 없이 enable까지 자동 진행되어 startup safety 설명이 약하다.

Need: startup state machine + arm contract

P1. Output capability mismatch

constants에는 1 MHz가 있지만 실제 `pulse_control.c`는 100 kHz clamp, bench discussion에서는 500 kHz fixed move를 원한다. 계약을 하나로 정리해야 한다.

Need: single actuator contract

P2. Fail-safe traceability gap

ESTOP, comm timeout, sensor fault, startup inhibit, recovery clear 조건이 fault taxonomy와 event log로 완전히 묶여 있지 않다.

Need: latched fault + clear policy + timestamps
